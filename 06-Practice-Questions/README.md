## Compilation Guide

### Basic Compilation
```bash
gcc program.c -o program
```

### With Warnings
```bash
gcc -Wall -Wextra program.c -o program
```

### With Math Library
```bash
gcc program.c -o program -lm
```

### With Debugging Info
```bash
gcc -g program.c -o program
```

### Optimized
```bash
gcc -O2 program.c -o program
```

### 1. Basic Testing

## Tips for Success

### 1. Problem-Solving Strategy
1. **Understand:** Read problem multiple times
2. **Plan:** Think before you code
3. **Implement:** Write clean code incrementally
4. **Test:** Verify with multiple test cases
5. **Optimize:** Improve if needed
6. **Review:** Learn from solutions

### 2. Debugging Techniques
- Use printf for tracing
- Check boundary conditions
- Verify pointer values
- Test with simple inputs first
- Use debugger for complex issues

### 3. Code Quality
- Use meaningful variable names
- Add comments for complex logic
- Follow consistent indentation
- Break complex functions into smaller ones
- Avoid magic numbers (use constants)

### 4. Common Pitfalls
- Array index out of bounds
- Uninitialized variables
- Memory leaks
- Integer overflow
- Off-by-one errors
- Forgetting null terminator in strings
